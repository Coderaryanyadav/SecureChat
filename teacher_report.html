Project Title: Secure Real-time E2EE Chat System

1. Project Overview

SecureChat is a modern web application designed for high-privacy communication. It allows users to create private "rooms" protected by a password. All messages are encrypted locally in the browser before being sent, ensuring that even the server cannot read the conversation content.

2. Key Features

  • End-to-End Encryption (E2EE): Uses the AES-GCM algorithm to secure messages.
  • Real-time Communication: Built using WebSockets for instant message delivery.
  • Dynamic Room Creation: Rooms are generated on-the-fly with unique keys.
  • Zero Data Retention: Messages are ephemeral and never stored in a database.

3. Technology Stack

  • Frontend: HTML5, CSS3 (Custom Design), JavaScript (ES6+).
  • Backend: Python 3.14, FastAPI Framework, Uvicorn Server.
  • Security: Web Crypto API (SubtleCrypto) for browser-based encryption.

4. Security Architecture

The project implements a "Blind Relay" architecture. When a user sends a message, the JavaScript client encrypts it using a 256-bit key derived from the room password. The server (FastAPI) receives this encrypted blob and broadcasts it to other members of the room. The server does not have the key and therefore sees only garbled text.

5. Conclusion

This project successfully demonstrates the implementation of secure web protocols and real-time data handling in a modern Python ecosystem. It prioritizes user privacy and efficient resource management.
